# Prompt για Claude — Pson.io App Improvements (Round 2)

Βοηθάς στη βελτίωση μιας Android εφαρμογής που ονομάζεται **Pson.io**.

**Tech stack:** React + Vite + TypeScript + Capacitor + shadcn-ui + Tailwind CSS  
**Repo:** https://github.com/er388/smart-shopper-pal  
**Package ID:** io.pson.app  
**Target:** Android 12+ (minSdk 26)  
**Εργαλεία:** Android Studio + GitHub Desktop  
**Τρέχουσα έκδοση:** 1.2.2

---

## ΚΑΝΟΝΕΣ — ΔΙΑΒΑΣΕ ΠΡΩΤΑ

**Κανόνας 1 — Έρευνα πριν από κάθε υλοποίηση — ΑΠΑΡΑΒΑΤΟΣ**  
Για ΚΑΘΕ εργασία, ψάξε πρώτα στο internet για τις καλύτερες πρακτικές από έμπειρους developers. Βάσισε την προσέγγισή σου σε αυτό που βρεις.

**Κανόνας 2 — Μην παίρνεις αποφάσεις μόνος σου**  
Για κάθε θέμα υλοποίησης, παρουσίασε τις επιλογές με τα trade-offs και ρώτησέ με πώς να προχωρήσουμε. Η μόνη εξαίρεση: αν σου πω ρητά "διάλεξε εσύ την καλύτερη λύση".

**Κανόνας 3 — Έλεγχος μετά από κάθε αλλαγή**  
Μετά από κάθε εργασία, επιβεβαίωσε ότι η εφαρμογή συνεχίζει να χτίζεται και λειτουργεί κανονικά. Αν κάτι κινδυνεύει να σπάσει, προειδοποίησέ με πριν το κάνεις.

**Κανόνας 4 — Semantic versioning**  
Το version number ανεβαίνει σύμφωνα με το semver.org: bug fix = patch, νέο feature = minor, breaking change = major. Ενημέρωσε και το `versionCode` και το `versionName` στο build.gradle.

**Κανόνας 5 — Ανθρώπινος κώδικας**  
Ο κώδικας πρέπει να φαίνεται γραμμένος από άνθρωπο developer. Αφαίρεσε ό,τι θυμίζει AI: υπερβολικά σχόλια που εξηγούν το προφανές, verbose ονόματα, περιττά checks, cookie-cutter δομή. Ελάχιστο αποτύπωμα σε μέγεθος κώδικα και χρήση RAM.

**Κανόνας 6 — Φάσεις εργασίας**  
Ομαδοποίησε τις εργασίες σε λογικές φάσεις. Ολοκλήρωσε και δοκίμασε κάθε φάση πριν ξεκινήσεις την επόμενη. Πρότεινέ μου πρώτα την ομαδοποίηση.

**Κανόνας 7 — Μία φάση τη φορά**  
Σε αυτή τη συνομιλία θα δουλέψουμε μόνο τη φάση που αναφέρεται στο τέλος αυτού του μηνύματος. Οι υπόλοιπες φάσεις θα δοθούν σε επόμενες συνομιλίες. Μην προτρέχεις. Όταν ολοκληρωθεί η φάση και επιβεβαιωθεί ότι λειτουργεί, ενημέρωσέ με και περίμενε οδηγίες για το επόμενο βήμα.

**Κανόνας 8 — Χωρίς λάθη**  
Δώσε μεγάλη προσοχή στο να μην υπάρχουν λάθη. Δίνε πλήρεις και σαφείς οδηγίες βήμα προς βήμα για κάθε αλλαγή. Η εφαρμογή πρέπει να είναι κατανοητή από οποιονδήποτε έχει χρησιμοποιήσει smartphone.

## ΕΡΓΑΣΙΕΣ

### Εργασία 1 — Κατάλογος στις Ρυθμίσεις
**α)** Στις Ρυθμίσεις, ο κατάλογος προϊόντων (`ProductCatalog` button) φαίνεται ξεκάρφωτος. Τα υπόλοιπα collapsibles (Καταστήματα, Κατηγορίες, Templates) έχουν συγκεκριμένο στυλ με περίγραμμα που επεκτείνεται. Ο κατάλογος πρέπει να ακολουθεί το ίδιο collapsible στυλ και να ελεγχθούν τα κενά μεταξύ όλων των sections.  
**β)** Όταν πατιέται ο κατάλογος τώρα, μεταφέρει στο πρώτο tab και ανοίγει εκεί σελίδα. Το επιθυμητό behavior: να ανοίγει ένα modal παράθυρο (overlay) όπου γίνονται όλες οι αλλαγές του καταλόγου, χωρίς αλλαγή tab.  
**γ)** Μέσα σε αυτό το modal, τα αντικείμενα που πατιούνται με αστεράκι ΔΕΝ πρέπει να πηγαίνουν αυτόματα στην κορυφή. Να υπάρχει drag-and-drop αναδιάταξη.

### Εργασία 2 — Drag-and-drop κατηγορίες προϊόντων
Να προστεθεί drag-and-drop αναδιάταξη και στις κατηγορίες προϊόντων στις Ρυθμίσεις (όχι μόνο στα custom, αλλά στα default επίσης). Η σειρά που ορίζεται εδώ να αντικατοπτρίζεται παντού στην εφαρμογή όπου εμφανίζονται κατηγορίες προϊόντων. Επίσης η σειρά που ορίζεται εδώ για τα προϊόντα να αντικατοπτρίζεται παντού στην εφαρμογή όπου εμφανίζονται προϊόντα.


### Εργασία 3 — Back button exit dialog
Το "Θέλετε να κλείσετε την εφαρμογή;" εμφανίζεται και σε άσχετες στιγμές — π.χ. κατά το editing μιας κατηγορίας και πάτημα back. Να εμφανίζεται **αυστηρά** μόνο όταν ο χρήστης είναι σε μία από τις 4 κεντρικές καρτέλες (/, /catalog, /history, /stats, /settings) **και** δεν γίνεται εκείνη τη στιγμή κάποιο edit. Επίσης υπάρχει bug: το dialog εμφανίζεται ως layer κατά την επεξεργασία προϊόντος και όταν γίνει tap, πατιέται ό,τι υπάρχει από κάτω χωρίς να φεύγει το dialog. Να διορθωθεί.

### Εργασία 4 — Keyboard language per field
Όταν ο χρήστης πατά ένα πεδίο:
- Πεδίο ελληνικού ονόματος → πληκτρολόγιο σε Ελληνικά
- Πεδίο αγγλικού ονόματος → πληκτρολόγιο σε Αγγλικά
- Πεδίο αριθμού/τιμής/ποσότητας → αριθμητικό πληκτρολόγιο
Μικρόφωνο (voice input) να υπάρχει και στο αγγλικό πεδίο, όχι μόνο στο ελληνικό. Αυτό πρέπει να αντικατοπτριστεί σε ΟΛΑ τα input fields της εφαρμογής. Ψάξε τα σωστά `inputMode` / `lang` attributes για React + Android WebView.

### Εργασία 5 — Scroll conflict στη γρήγορη προσθήκη
Στη λίστα αγορών, όταν κάνω scroll οριζόντια στη γρήγορη προσθήκη (Quick Add) για να δω αντικείμενα πιο δεξιά, μόλις σηκωθεί το δάκτυλο αλλάζει η οθόνη (ο swipe navigation ερμηνεύει την κίνηση ως tab change). Να διορθωθεί ώστε το swipe εντός του Quick Add container να μην προκαλεί αλλαγή tab. Ψάξε για `touch-action`, `pointer events` και event propagation solutions στο Capacitor/React context.

### Εργασία 6 — Store selection στην ολοκλήρωση αγοράς
Στο dialog "Δεν έχετε επιλέξει κατάστημα" που εμφανίζεται κατά την ολοκλήρωση αγοράς, το "Επιλογή καταστήματος" να γίνει dropdown με τα διαθέσιμα καταστήματα. Στην κορυφή του dropdown να υπάρχει η επιλογή "Συνέχεια χωρίς κατάστημα".

### Εργασία 7 — Αλλαγή καταστήματος με αντικείμενα στη λίστα
Όταν υπάρχουν αντικείμενα στη λίστα και αλλάξει το ενεργό κατάστημα:
- Τα αντικείμενα του προηγούμενου καταστήματος να αποθηκεύονται ως "υπόλοιπα" σε εκείνο το κατάστημα.
- Να υπάρχει επιλογή import για να φορτωθούν τα υπόλοιπα από άλλο κατάστημα ως εκκρεμότητες.
- Αν κάποιο αντικείμενο αγοραστεί από άλλο κατάστημα, να διαγραφεί από το πρώτο.
Ψάξε πώς άλλες εφαρμογές λιστών χειρίζονται multi-store shopping πριν προτείνεις implementation.

### Εργασία 8 — Overflow στο barcode scanner
Κατά το σκανάρισμα, αν αναγνωριστεί κάτι πολύ μεγάλο (π.χ. QR code με πολλές πληροφορίες), το αποτέλεσμα "φεύγει" από τη δεξιά μεριά της οθόνης. Να υπάρχει όριο χαρακτήρων στο display και να γίνεται αναδίπλωση (word-wrap) του κειμένου.

### Εργασία 9 — Category dropdown χωρίς διπλό tap
Στη φόρμα νέου/επεξεργασίας προϊόντος, όταν ο χρήστης γράφει κάτι σε πεδίο και μετά πατά κατ' ευθείαν το dropdown της κατηγορίας, σβήνει το πληκτρολόγιο χωρίς να ανοίγει το dropdown — χρειάζεται δεύτερο tap. Να ανοίγει κατ' ευθείαν με ένα tap. Ψάξε για `onPointerDown` vs `onClick` solutions για αυτό το pattern.

### Εργασία 10 — Εξαγωγή δεδομένων: επιλογή τοποθεσίας
Η εξαγωγή δεδομένων (JSON backup) να ρωτά τον χρήστη πού να αποθηκευτεί το αρχείο στη συσκευή. Ψάξε για το Capacitor Filesystem plugin και το Android Storage Access Framework για την υλοποίηση.

### Εργασία 11 — Share: native Android panel
Το share button γράφει μόνο "Αντιγράφηκε" αντί να ανοίγει το native Android share sheet. Ψάξε τη σωστή υλοποίηση με το Capacitor Share plugin και την `navigator.share()` API για να ανοίγει το native share panel.

### Εργασία 12 — App icon tappable + about popup
Το εικονίδιο μέσα στην εφαρμογή (στο Settings, στο splash screen) να γίνει το ίδιο με αυτό που εμφανίζεται στο τηλέφωνο (launcher icon). Να είναι tappable και να ανοίγει ένα popup παράθυρο που εμφανίζει το README — το popup να είναι scrollable και να έχει κουμπί X για κλείσιμο. Ψάξε στο internet για best practices για αυτό το "About" button pattern σε mobile apps και πρότεινέ μου επιλογές πριν υλοποιήσεις.

### Εργασία 13 — Έρευνα & προτάσεις βελτιώσεων
Ψάξε στο internet για: (α) δημοφιλείς features σε ανταγωνιστικές εφαρμογές λίστας ψωνιών (Listonic, OurGroceries, AnyList, Bring!), (β) UX patterns που λείπουν από την εφαρμογή, (γ) performance optimizations για React + Capacitor apps. Παρουσίασέ μου τα ευρήματα ως λίστα προτάσεων με σύντομη αιτιολόγηση για κάθε μία. Εγώ αποφασίζω τι θα υλοποιηθεί.
